What Git I know

  1. Cloning: Using git clone to clone my repositories from GitHub to my local machine.
  2. Status: Using git status to check what files have been modified.
  3. Add: Using git add . or git add file name to add the changes or codes I've added.
  4. Commit: Using git commit -m "commit header" -m "commit details" to commit what I've added.
  5. Push: Using Git push origin main to push the changes or codes I've added to GitHub making it update the live website.
  6. Branching: Using git branch feature-name to create a new branch so I can make and test changes without breaking the live website. Using Git checkout feature-name to access the new branch.
  7. Merging: Using git merge feature-name to merge the branched version to the main version once I am happy with it. Using Git checkout origin main to go back to the main repo, ready for it to be commited and pushed.
